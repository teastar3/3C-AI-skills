

# 3C产品经理

**7 个 AI Skill，覆盖 3C 产品从 0 到 1 的全流程。**

做 3C 产品最难的事，不是写代码，而是想清楚——供应链、BOM 成本、渠道利润、硬件体验、用户场景，每一步都是坑。

3C 产品大师 Pro 是一组结构化的 AI Prompt（Skill），把 3C 产品经理从"定义硬件需求"到"传播硬件策略"的全流程，拆成了 **6 个独立的硬件分析模块 + 1 个编排模块**。每个模块背后有具体的硬件方法论支撑——不是空泛的框架，而是能直接用的硬件工具。

```
                    ┌─────────────────┐
                    │   ⑦ 3C产品圆桌   │  ← 编排层：串联全流程
                    └──────┬──────────┘
                           │
        ┌──────┬──────┬──────┬──────┬──────┐
        ①      ②      ③      ④      ⑤      ⑥
     定义3C  理解3C  3C产品  3C竞争  3C执行  3C传播
     产品需求 用户场景 做不做  分析   策略   策略
```

可以单独调用任何一个 Skill，也可以用「3C 产品圆桌」一键串联全流程。

3C 产品经理的日常大概是这样的：

- 周一 ，老板说"我想做一个智能硬件"。你点头说好，但心里没底——这个"硬件"到底要解决什么问题？BOM 成本多少？
- 周二 ，做了份用户调研。100个人说"挺好的，我会买"。但你知道，问卷上说"会买"和真的掏钱是两回事。
- 周三 ，竞品分析写了20页PPT。结论是"差异化竞争"。但这四个字约等于什么都没说。
- 周四 ，硬件方案列了30个功能。每个都有人说重要。你不知道该砍哪个，也不知道哪个会爆 BOM 成本。
- 周五 ，市场部要广告语。"极致体验，尊享品质"——说出来连你自己都不信。

**这些问题之间有逻辑顺序**：想不清楚目标，就看不懂用户；看不懂用户，就不知道做不做；不知道做不做，竞争分析就是自嗨；分析再好，供应链搞不定也是零；最后好不容易做出来了，用户记不住你。

每一步断裂，后面全部白做。

# Skill 概览

| Skill | 解决什么 | 核心方法论 |
|-------|----------|------------|
| ① **定义3C产品需求** | 把模糊硬件想法变成具体目标 | 封闭目标五要素、技术规格定义、硬件可行性检查、存在感公式 |
| ② **理解3C用户场景** | 从硬件使用场景挖出深层动机 | 五层硬件画像、使用场景重构、力场分析、贪嗔痴 |
| ③ **3C产品做不做** | 算硬件账，值不值得投入 | 硬件三问、用户价值公式、BOM成本、渠道利润、ROIC |
| ④ **3C竞争分析** | 硬件护城河、对手、博弈 | 硬件护城河三重测试、博弈论、EPV、特许经营区边界 |
| ⑤ **3C执行策略** | 砍功能、出计划、先验证 | 杀功能、极简规则、小白思维、预型测试、硬件MVP |
| ⑥ **3C传播策略** | 造一句顾客替你传播的硬件口号 | 华与华·播传理论、四大算法、邻居测试、100年测试 |
| ⑦ **3C产品圆桌** | 一键全流程硬件诊断 | 编排①→⑥ + 矛盾检测 + 大师辩论 |

## 使用方法

1. 打开任意一个 `skill-*.md` 文件
2. 将其内容作为 System Prompt 粘贴到你的 AI 对话中
3. 输入触发词开始使用

**触发词示例**：

| Skill | 触发词 |
|-------|--------|
| 定义3C产品需求 | `定义3C产品需求`、`帮我想清楚硬件`、`我想做一个XX硬件` |
| 理解3C用户场景 | `理解3C用户场景`、`硬件用户洞察`、`用户为什么会用这个硬件` |
| 3C产品做不做 | `3C产品做不做`、`硬件值不值得做`、`帮我算硬件账` |
| 3C竞争分析 | `3C竞争分析`、`硬件护城河`、`硬件市场能不能进` |
| 3C执行策略 | `3C执行策略`、`硬件怎么落地`、`硬件MVP怎么做` |
| 3C传播策略 | `3C传播策略`、`帮我写硬件文案`、`硬件品牌口号` |
| 3C产品圆桌 | `3C产品圆桌`、`完整硬件诊断`、`帮我从头到尾看看硬件` |

## 场景匹配

| 场景 | 用哪个 Skill |
|------|--------------|
| 只想验证一个硬件想法靠不靠谱 | ③ 3C产品做不做 |
| 想深入了解目标用户怎么用硬件 | ② 理解3C用户场景 |
| 硬件做好了，需要一句广告语 | ⑥ 3C传播策略 |
| 有个完整硬件想法，想全面诊断 | ⑦ 3C产品圆桌 |
| 已做完定义需求，想继续下一步 | 从任意一步开始，按需组合 |

每个 Skill 都能独立运行，也能被 3C 产品圆桌串联调用。像乐高积木一样——单块有用，拼起来更强。

## 方法论来源

这组 Skill 不是凭空编的。每一个工具都有明确的理论出处，并针对 3C 硬件特点进行了特化：

| 来源 | 贡献 | 3C 特化 |
|------|------|---------|
| **傅盛** | 封闭目标、破局点、All in | 硬件破局点检查、供应链聚焦 |
| **雷军** | 性价比、供应链、品质 | BOM 成本控制、供应链管理、品质标准 |
| **俞军** | 用户价值公式、产品即交易 | 硬件用户价值、替换成本拆解 |
| **张小龙** | 极简规则、产品演化、用完即走 | 硬件极简规则、小白可用性 |
| **周鸿祎** | 小白思维、微创新、单点突破 | 硬件小白测试、硬件微创新点 |
| **曹政** | 流量公式、信息差、先人工、看日志 | 硬件渠道成本、硬件数据监控 |
| **Bruce Greenwald** | 护城河三重测试、EPV | 硬件护城河、供应链壁垒 |
| **华与华** | 播传理论、文化母体 | 硬件品牌谚语、硬件口语化 |
| **Alberto Savoia** | 预型测试（Pretotyping） | 硬件预型测试、3D 打印样机 |

不是搬运理论，是把这些理论变成了**一步步可执行的硬件操作流程**。
